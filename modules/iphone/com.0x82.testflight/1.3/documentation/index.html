testflight Module

Description

This Titanium module wraps the oficial Testflight SDK, available at https://testflightapp.com/sdk/.

It allows you to write Titanium apps and still benefict from using the Testflight SDK.

Accessing the testflight Module

To access this module from JavaScript, you would do the following:

var testflight = require("com.0x82.testflight");

The testflight variable is a reference to the Module object.

Reference

testflight.takeOff(teamToken)

This should be the first function you should call to initialize the module. It starts a TestFlight session, setups the crash handler, and sends pending notifications to the server.

It accepts a single argument teamToken, with your TestFlight team token. You can obtain that token at https://testflightapp.com/dashboard/team/edit

testflight.passCheckpoint(checkPointName)

Track when a user has passed a checkppint after the flight has taken off. Eg. passed level 1, posted hight score, etc.

It accepts a single argument checkPointName with the String name of the checkpoint.

testflight.openFeedbackView()

Opens a feedback window that is not attached to a checkpoint. You should just call it with no arguments. The windows open modally.

testflight.addCustomEnvironmentInformation(hashOptions)

Add custom environment information. If you want to track a user name from your application you can add it here.

It accepts a single hash/dictionary argument hashOptions, with key/value pairs of options you want to set on your environment.

testflight.submitFeedback(text)

Sends a custom feedback to the server without showing the feedback view. Accepts a single parameter which should be a String with the text to send to the testflight server.

testflight.setDeviceIdentifier(string)

The module no longer obtains the device unique identifier. This method should only be used during testing so that you can identify a testers test data with them. If you do not provide the identifier you will still see all session data, with checkpoints and logs, but the data will be anonymized. It is recommended that you only use this method during testing.

TL;DR: Using this method on production could make you app fail the approval process!
